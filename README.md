# AI SQL Telegram Bot
Данный telegram-бот выдает статистику по видео-креаторам публикующим контент в разных соц. сетях. База данных содержит информацию о видеороликах (id видео, id креатора, дата и время публикации видео, общее количество просмотров, лайков, комментариев и жалоб), а также информацию о почасовых снапшотах каждого видео (id снапшота, время снапшота, текущие значения количества просмотров, лайков, комментариев и жалоб на момент снапшота, приращения количества просмотров, лайков, комментариев и жалоб с момента прошлого снапшота). Ответ подразумевается в виде одного числа.
## Запуск проекта локально
Для запуска проекта локально необходимо:

 - Скачать данный репозиторий
 - На основе файла *.env_example* создать файл *.env* и добавить в него токен бота и ключ для нейросети gigachat
 - Установить Docker Desktop
 - Запустить контейнеры командой **`docker-compose up`**

## Получение токена бота и ключа для нейронной сети
 

 - Токен бота можно получить после его создания через бота @BotFather. 
- Как получить ключ для gigachat можно узнать на официальном сайте: https://developers.sber.ru/docs/ru/gigachat/quickstart/ind-using-api?tool=python#poluchenie-avtorizatsionnyh-dannyh
 
 
## Архитектура и подход к преобразованию текстовых запросов в обращения к БД

 Архитектура проекта:
```
📂 project
 ├── 📁 dbserver
 │   ├── 📁 initdb.d
 │   │   └── 📄 load_json.sql
 │   └── ⚙️ videos.json
 ├── 📁 tgbot
 │   ├── 📁 handlers
 │   │   ├── 📄 ai_handlers.py
 │   │   └── 📄 basic_handlers.py
 │   ├── 📄 main.py
 │   └── 📄 requirements.txt
 ├── ⚙️ .dockerignore
 ├── ⚙️ .env
 ├── ⚙️ .env_example
 ├── ⚙️ .gitignore
 ├── ⚙️ docker-compose.yml
 ├── ⚙️ Dockerfile_dbserver
 └── ⚙️ Dockerfile_tgbot
```
Проект состоит из двух docker-контейнеров:
- Один для хранения базы данных PostgreSQL 18.2
- Второй для python-приложения telegram бота

С помощью docker-compose оба контейнера собираются и запускаются одной командой: 
```
docker-compose up
```
Принцип работы telegram бота:

 - Вопрос telegram боту задается на русском (естественном) языке
 - После этого он передается нейросети gigachat для преобразования в SQL-запрос
 - SQL-запрос выполняется в СУБД PostgreSQL
 - Ответ СУБД в виде одного числа отправляется пользователю

Используемая для запросов к LLM схема данных описана в промпте, который представлен в файле *ai_handlers.py*.
```
 📁 tgbot
 └── 📁 handlers
      └── 📄 ai_handlers.py
```
Исходная информация для СУБД подгружается из файла *videos.json* с помощью скрипта *load_json.sql* во время инициализации базы данных. Предварительно файл *videos.json* на этапе сборки очищается от переносов строк и пробелов.
```
📁 dbserver
 ├── 📁 initdb.d
 │   └── 📄 load_json.sql
 └── ⚙️ videos.json
 ```
